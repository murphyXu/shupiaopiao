# 书架与漂流首页布局对齐设计

**日期：** 2026-06-21

**状态：** 已确认方案 A

## 目标

在不改变现有数据、事件和导航行为的前提下，统一书架首页与漂流首页的顶部信息层级，并解决书架标题区操作入口分散的问题。

## 书架首页

### 标题操作行

- 书架名称、`编辑`、`分享书架` 位于同一行。
- `编辑` 与 `分享书架` 使用独立的等高文字按钮，视觉尺寸一致、点击区域明确。
- 两个按钮紧邻书架名称右侧，不再把分享按钮单独推到页面最右侧。
- 分享模式不展示编辑按钮；分享按钮继续保留现有 `open-type="share"` 行为。
- 编辑状态继续用现有输入框和保存按钮替换标题行，不改变保存逻辑。

### 容量操作行

- `剩余可收藏 N 本 / 上限 N 本` 与 `兑换额度` 位于同一行。
- `兑换额度` 紧邻容量文案右侧并垂直居中，不再单独占据下一行。
- 未登录或分享模式不展示兑换入口，保持现有权限逻辑。
- 容量数字和兑换页面导航行为不变。

### 小屏适配

- 标题名称区域允许收缩，书架名称使用单行省略，保证两个操作按钮不被挤出屏幕。
- 操作按钮禁止压缩，按钮文字保持单行。
- 容量文案可收缩并单行省略，兑换按钮保持完整可点击。

## 漂流首页

### 模块顺序

漂流首页顶部模块严格对齐书架首页：

1. 页面标题与副标题。
2. 三项数据统计。
3. 一级分类导航。
4. 二级分类导航。
5. 搜索框。
6. 漂流图书列表。

当前漂流页位于统计模块之后的搜索框移动到二级分类导航之后。

### UI 对齐

- 数据统计继续复用全局 `stat-row` / `stat-card` 样式。
- 一级分类导航的圆角、背景、间距、激活态与书架首页一致。
- 二级分类导航的滚动容器、标签高度、间距、边框和激活态与书架首页一致。
- 漂流页 `.pool-search-box` 的声明与书架页 `.shelf-search` 同类规则一致，但不跨页面复用 class。
- 漂流页保留自己的搜索占位文案和输入事件，不改变筛选、查询及列表滚动逻辑。

## 实现边界

- 仅修改 `miniprogram/pages/shelf/index.wxml`、`miniprogram/pages/shelf/index.wxss`、`miniprogram/pages/pool/index.wxml`、`miniprogram/pages/pool/index.wxss` 及相关合同测试；同步更新 `scripts/test-shelf-interaction-contract.js` 中旧的容量裁剪合同、`scripts/test-pool-want-contract.js` 中“搜索在筛选前”的旧顺序合同，以及 `scripts/test-pool-navigation-and-anonymous-contract.js` 中跨页面复用 `shelf-search` 的旧断言。
- 不修改云函数、接口、页面 JavaScript 状态结构或导航逻辑。
- 不抽取新的全局样式，避免影响其他已使用同名类的页面。

## 验证

- 合同测试验证书架标题行同时包含书架名、编辑和分享入口，并验证两个按钮共享等高样式。
- 合同测试验证容量文案与兑换入口位于同一容器。
- `scripts/test-shelf-interaction-contract.js` 同步验证容量文案在小屏使用 `overflow: hidden` 与 `text-overflow: ellipsis`，不再保留“不得裁剪/省略”的旧断言。

- 合同测试验证漂流页的一级、二级导航和搜索框完整规则声明与书架首页一致。

非 UI / 交互层面的逻辑改动清单

列表与筛选逻辑

  • 漂流广场默认从“三种筛选模式”改为“推荐 + 高频类目”，复杂筛选收进面板。
  • 新增“推荐”排序策略：可优先同城/近距离、有封面、可接漂、信用良好、轻量书。
  • 保留原有品类、积分、品相筛选，但入口和状态管理需要调整。

接漂申请逻辑

  • 申请页需要聚合展示地址、可用公益积分、占用积分、寄出地、未收货上限。
  • 确认按钮状态需要依赖地址是否存在、公益积分是否足够、是否触达 5 单上限。
  • 积分不足时建议从“直接失败”前移为“主按钮降级 + 引导获得积分 / 加入想要”。

上漂发布逻辑

  • 发布页需要步骤状态：选书、品相、积分、确认，但可继续单页提交。
  • 定价缺失应在选择阶段阻断或强提示，不建议提交时静默自动取消。
  • 书架卡片直达上漂时，需要携带 shelfBookId 并预选对应书籍。

书架供给逻辑

  • 书架新增“可上漂 / 已上漂”视图，需要使用 canPublishDrift、漂流状态做筛选。
  • 书卡按钮根据状态切换：上漂这本、查看漂流、不可上漂原因。
  • 容量不足时添加书籍入口先提示兑换额度，不进入无效添加流程。

我的与订单逻辑

  • 我的页待办模块需要按状态聚合并支持点击直达订单列表对应状态。
  • 订单卡新增 nextStep 文案字段，按赠出方/接漂方视角计算下一步。
  • 合并发货订单要显示 orderCount、同一收件地址、合并发货入口。

积分明细逻辑

  • “总记录值”文案改为“累计记录”，展示逻辑仍来自现有 balance。
  • 交易流水增加分类视图：全部、增加、占用、释放、扣除。
  • 流水描述需要统一为公益积分语义，继续避免钱包、余额、购买式表达。

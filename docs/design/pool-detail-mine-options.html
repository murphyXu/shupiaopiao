漂流详情 ·「我的赠书」弱化方案

当前问题：底部灰色固定按钮看起来像可点的主操作，但实际无 bindtap，且与列表角标重复。 他人书籍的底栏是「想要 + 申请接漂」决策区；自己的书不需要决策，不宜占用同级视觉权重。

现状：固定底栏 + 360rpx 宽灰按钮「我的赠书」+ 页面底部留白 200rpx —— 占位大、像假按钮。

方案 A · 去掉底栏，首卡内状态推荐

封面角标（与列表一致）+ 标题区一行状态文案，无固定底栏。

注意
需减 padding-bottom；可链到赠出记录

浅灰信息条替代底栏按钮，明确是「说明」不是「操作」。

组合建议

  • 首选 A + D：封面角标 + 首卡一行状态 + 页末「查看赠出进度」弱链接；彻底去掉固定底栏。

  • 不推荐：保留 C 的底栏细条（仍像「有个底栏」）；或继续用按钮形态哪怕改灰色。

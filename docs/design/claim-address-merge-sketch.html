结构与交互

wxml 结构（已有地址）
  1. address-card 单卡片，不再渲染下方 wx-address-btn
  2. 标题行：收货地址 + 修改地址 ›（catchtap="selectAddress" 或共用父级 tap）
  3. 正文区：姓名电话 + 省市区详细地址（点击同样 selectAddress）
  4. 删除独立「读取微信地址」按钮；微信导入改在地址簿页或空态提供
点击行为
  1. 点击「修改地址 ›」→ 地址簿选择页（现有 selectAddress）
  2. 点击地址正文 → 同上
  3. 从地址簿返回后刷新 shippingHint（现有逻辑不变）
文案
  1. 有地址：右侧 修改地址 ›
  2. 无地址：右侧 读取微信地址（方案 A）或卡片内底栏（方案 B）

样式参考（对齐 profile 页 mini-link）

复用 pages/auth/profile 的 .section-head + .mini-link 模式： 左标题、右品牌绿文字链，与「玩法介绍 ›」「查看漂流玩法 ›」视觉一致。
